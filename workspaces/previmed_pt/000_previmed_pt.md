# Previmed.pt

Workspace do site Previmed.pt.

## Objetivo
Representar o trabalho concreto no site **Previmed.pt** — desenvolvimento, conteúdo, SEO, performance e conversão.

## O que vai guardar aqui
- contexto e estado do site Previmed.pt;
- frentes de trabalho (páginas, conteúdo, SEO, performance);
- decisões e notas específicas do site.

## Departamentos relacionados
- `web` (principal) — desenvolvimento, WordPress, SEO, performance, UX/UI.
- `communications` — conteúdo e linguagem.
- `commercial` — páginas e conversão comercial.
- `compliance` — dados de formulários e RGPD.
- `health_safety` — conteúdo técnico do serviço.

## Tools relacionadas
Ver [tools.md](tools.md) deste workspace.

## Modules relacionados
Ver [modules.md](modules.md). Este workspace usa o module **`seo-growth-system`** para conteúdo, pesquisa orgânica, WordPress SEO, performance web, schema, local SEO, concorrência e AI search. SEO entra via **SEO Lead** do module (ou `/seo`), só para tarefas web/search/content. WordPress, visual, performance e compliance entram conforme o caso.

## Como os agentes vão usar
Quando o pedido é sobre o site Previmed.pt, o agente entra por aqui, identifica os departamentos relevantes, usa as tools de `tools.md` e os modules de `modules.md` (SEO via SEO Lead).
