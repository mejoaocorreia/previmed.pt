# Manuals — Compliance

Procedimentos de compliance da Previmed.

## Objetivo
Reunir o *como se faz* de RGPD, dados sensíveis, permissões e risco. Procedimentos frequentemente **transversais**.

## Que procedimentos entram aqui
- tratamento de dados pessoais e RGPD;
- pedidos de acesso e gestão de permissões;
- avaliação e registo de risco.

## Departamentos relacionados
- `compliance` (principal); transversal a todos.

## Workspaces relacionados
- todos os que tratam dados (ex.: `careview`, formulários do `previmed_pt`).

## Como os agentes vão usar
Sempre que há dados sensíveis/RGPD/risco, o agente consulta estes procedimentos antes de avançar. Articula com `.claude/project/system-safety`.
