# Web

Departamento web da Previmed.

## Objetivo
Cobrir tudo o que é web — **mais amplo do que websites**: sites, web development, WordPress, SEO, performance, UX/UI, landing pages e integrações.

## O que vai guardar aqui
- desenvolvimento e manutenção de sites;
- WordPress (temas, plugins, segurança);
- SEO / organic growth (capacidade, não identidade do repo);
- performance e Core Web Vitals;
- UX/UI e landing pages;
- integrações web.

## SEO Growth System (module)
O SEO é uma **capacidade do departamento web**, organizada como **module reutilizável**: [`.claude/modules/seo-growth-system/`](../../.claude/modules/seo-growth-system/README.md). Deve ser chamado para workspaces web (ex.: Previmed.pt) via **SEO Lead** do module ou comando `/seo`, e só quando o pedido tocar web/search/content/WordPress SEO. Sendo um module, é reutilizável noutros projetos, não está preso à Previmed.

## Como os agentes vão usar
Quando o pedido toca **web, search ou conteúdo digital**, o agente entra por aqui. Para SEO, encaminha para o module `seo-growth-system` (SEO Lead). A implementação WordPress de risco articula com WordPress Engineering e o supervisor.

## Relação com workspaces
Serve diretamente o workspace Previmed.pt e dá suporte web a outros workspaces (ex.: páginas/integrações do Careview).
