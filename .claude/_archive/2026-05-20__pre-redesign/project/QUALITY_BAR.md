# Quality Bar

## Objetivo

O resultado deve parecer profissional, premium, moderno e confiável.

Não aceitar:
- aspeto genérico;
- visual de template barato;
- hierarquia fraca;
- espaçamentos aleatórios;
- mobile quebrado;
- texto confuso;
- CTAs fracos;
- animações gratuitas;
- SEO ignorado;
- performance ignorada.

## Design

Avaliar sempre:

- hierarquia visual;
- ritmo;
- contraste;
- tipografia;
- coerência;
- alinhamento;
- espaçamento;
- consistência de botões/cards;
- clareza do CTA;
- legibilidade;
- confiança;
- mobile.

## SEO

SEO é parte da qualidade, não uma tarefa secundária.

Avaliar:
- H1 único;
- H2/H3 coerentes;
- conteúdo indexável;
- intenção de pesquisa;
- links internos;
- meta title;
- meta description;
- schema;
- performance;
- acessibilidade;
- estrutura de páginas.

## Performance

Avaliar:
- peso das imagens;
- JS/CSS carregado;
- fonts;
- lazy loading;
- Core Web Vitals;
- LCP/INP/CLS;
- mobile.

## Motion / 3D

Permitido quando acrescenta valor.

Requer:
- objetivo claro;
- fallback;
- teste mobile;
- respeito por prefers-reduced-motion;
- performance aceitável;
- não esconder conteúdo SEO.

## Aprovação

Uma secção/página não está pronta só porque “funciona”.
Deve estar:
- clara;
- bonita;
- rápida o suficiente;
- responsiva;
- semanticamente correta;
- coerente com a marca;
- fácil de manter.
