# Content Brief Example

## Content Brief

### Objetivo
Criar uma pagina de servico clara, confiavel e orientada a conversao.

### Publico-alvo
Decisor que procura entender o servico, requisitos e proximo passo.

### Intencao
Comercial / transacional.

### Page type
Pagina de servico principal.

### Outline
1. O que e o servico.
2. Para quem e.
3. Quando e necessario.
4. Como funciona.
5. Prova/confiança.
6. FAQs reais.
7. CTA.

### Prova necessaria
- Credenciais reais.
- Processo verificavel.
- Claims revistos por humano se forem sensiveis.

### Internal links
- Link para homepage.
- Link para pagina de contacto.
- Links de guias educativos relevantes.

### Schema candidate
Service + BreadcrumbList. Validar com `schema-entity`.

### QA risks
- Claims sem prova.
- Conteudo generico.
- CTA sem contexto.
